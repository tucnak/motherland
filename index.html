Motherland CSS

A lightweight CSS framework for the real men!

What's the Hell?

Motherland is a dead simple modern technology, which do reckon with the old school. We are damn pissed off by css hipsters and all the new sorts of crap appearing online every single day. Material design, huh? Bootstrap? Pure.css, whatever. We are men, not a bunch of creepy hippy. We want to deliver user some information without web3.0-ish maroon interface.

From the developer's perspective, Motherland is designed to be super intuitive, so the whole markup thing looks logical and clear for anyone looking at the source code of your project. You guys are fighting at the front of user experience, where little mistakes do matter. Your task is to create user interface that will feel smooth for the end user. Right now, go on and just ask yourself, would you want your users to get distracted with some hippy shit? — We don't.

Grid

We stick to the subdivisional grid layout system, which recommended itself as a solid solution for applications of different scale: starting with demonstration pages like this, finishing with complex web application. You are supposed to divide container with whole, half, thirds and fifths classes and corresponding fraction number. Here is a demo for fifths fractioning:

.one.fifth
.four.fifths
.two.fifths
.three.fifths
.three.fifths
.two.fifths
.four.fifths
.one.fifth

Each subdivided row is represented .unit block, containing fractional blocks. Each .unit block will take maximum space. We also prefer to not bother you guys with default paddings and margins, because they irritate everybody so much. Here is an example of complicated markup, which shows the point of combining different subdivisions:

.two.fifths

The main reason to use subdivisional system, not column-driven, comes from the point that subdivisions are much more native than columns approach. While developing the user interface, columns might appear distracting, while subdivisions feel smooth. A few words about .row class, that is common in most CSS frameworks around. We don't find it native at all, especially for complicated layout cases, that's why we label such block with a .unit class.
